# Hana Safe Lease Passport — 기능명세서
[[API_명세서]]

> 전세 계약 전 위험을 먼저 계산하는 디지털 주거금융 서비스  
> 작성 기준: MVP (Phase 1) 범위  
> v3 반영: 등기부 캐시/재사용 구조, 리포트 해설형 AI 확장, 이력 기록 보강

## 1. 서비스 구조 개요

| 구분 | 설명 |
|------|------|
| 서비스명 | Hana Safe Lease Passport |
| 플랫폼 | 하나원큐 앱 내 탑재 (모바일 우선) |
| 대상 사용자 | 전세 계약을 앞둔 20~30대 청년층 중심, 전 연령 이용 가능 |

| MVP 지역 범위 | 서울·수도권, 빌라·오피스텔 중심 |

## 3. 사용자 여정 (User Flow)

```
[매물 발견] → [서비스 진입] → [주소+보증금 입력] → [1차 기본분석] → [등기부 업로드(선택)] → [2차 정밀분석] → [리포트 발급] → [금융 액션 실행]
```

### F-03. 등기부등본 업로드 + AI 파싱

| 항목 | 내용 |
|------|------|
| 기능 ID | F-03 |
| 범위 | **MVP 필수** (업로드는 사용자 선택) |
| **입력** | 인터넷등기소 발급 등기부등본 PDF |
| **처리** | PDF → OCR 텍스트 추출 → NLP 구조화 → 위험 요소 자동 태깅 |
| **출력** | 구조화된 등기 데이터: 근저당 건수·금액, 가압류 유무, 신탁 여부, 소유권 이전 이력 |
| 추출 대상 | 갑구: 소유권 이전·가압류·가처분·신탁 / 을구: 근저당 설정 건수·금액 / 표제부: 용도·면적 |
| 미업로드 시 | 1차 기본 분석만 제공 (전세가율 + 보증 가입 가능성) |
| 파싱 실패 시 | "자동 인식에 실패했습니다" → 핵심 3항목(근저당 금액, 가압류 유무, 소유권 이전일) 수기 입력 |
| 비용 | 등기부 발급 비용(700~1,000원) 사용자 부담 |

#### 등기부 저장 및 재사용 정책

| 항목 | 내용 |
|------|------|
| 저장 방식 | 사용자 동의 하에 파싱된 등기 데이터를 암호화 저장 (원본 PDF는 파싱 완료 후 24시간 내 삭제) |
| 재사용 | 동일 매물 재조회 시, 저장된 등기 데이터를 재사용하여 추가 열람 비용 없이 분석 가능 |
| 유효기간 | 저장된 등기 데이터는 발급일 기준 30일간 유효. 만료 시 "등기부가 30일 이상 경과했습니다. 최신 등기부를 업로드해주세요" 안내 |
| 최신성 보장 | 유효기간 내라도 사용자가 수동으로 재업로드하면 기존 데이터를 최신 버전으로 갱신 |

> 이 구조는 API/열람 비용 절감 + 사용자 재방문 경험 개선 + "매번 다시 열람해야 하나요?"에 대한 답을 동시에 제공한다.

### F-05. AI 위험 분석 엔진

| 항목 | 내용 |
|------|------|
| 기능 ID | F-05 |
| 범위 | **MVP 필수** (핵심 3변수만) |
| **입력** | 주소, 보증금, 등기부 PDF(선택) |
| **처리** | 시세 조회 → 전세가율 계산 → 보증 가입 조건 판별 → (등기 업로드 시) 등기 위험 파싱 → 종합 판정 + 위험 노출 구간 산출 |
| **출력** | 변수별 위험도(🟢🟡🔴), 종합 판정, 위험 노출 구간(금액 범위), 추천 액션 리스트 |

#### MVP 분석 변수 (핵심 3변수)

| 변수 | 분석 내용 | 데이터 소스 | 기법 |
|------|----------|------------|------|
| ① 등기 권리관계 | 근저당·가압류·신탁, 소유권 변동 | 등기부등본 (사용자 업로드) + 건축물대장 API | OCR + NLP 문서 해석 + 룰 엔진 |
| ② 전세가율 적정성 | 시세 대비 보증금 비율 | 국토부 실거래가 API + 시세 데이터 | 수치 계산 + 지역별 기준 비교 |
| ③ 보증 가입 가능성 | HUG·HF 가입 조건 충족 여부 | 보증기관 공개 기준 | 룰 기반 조건 판별 |

> **MVP의 AI 핵심 = 문서 해석 + 규칙 기반 판단 + 수치 계산.**  
> 고급 예측 모델(Isolation Forest, GBM, GIS)은 Phase 2~3 고도화 단계에서 적용.

#### 고도화 변수 (Phase 2~3)

| 변수 | 단계 | 기법 |
|------|------|------|
| ④ 거래 패턴 이상 | Phase 2 | Isolation Forest |
| ⑤ 입지·유형 위험도 | Phase 2 | GBM 분류기 |
| ⑥ 기후 리스크 | Phase 3 | GIS 공간 매칭 |

#### 변수별 위험 등급 기준

| 변수 | 🟢 낮음 | 🟡 중간 | 🔴 높음 |
|------|--------|--------|--------|
| ① 등기 권리관계 | 근저당 1건 이하, 가압류·신탁 없음 | 근저당 2건+ 또는 소유권 이전 1년 이내 | 가압류 존재, 신탁 등기, 소유권 분쟁 |
| ② 전세가율 | 70% 이하 | 70~85% | 85% 초과 |
| ③ 보증 가입 가능성 | HUG/HF 조건 충족 | 일부 미충족 (보완 가능) | 가입 불가 |

#### 종합 판정 로직

| 판정 | 조건 |
|------|------|
| 🔴 신중 검토 권고 | 핵심 3변수(①②③) 중 하나라도 🔴 → 자동 🔴 |
| 🟡 보완 후 가능 | 🔴 없이, 🟡이 2개 이상 |
| 🟢 계약 가능 | 🔴 없고, 🟡이 1개 이하 |

#### 위험 노출 구간 계산 (등기부 업로드 시)

```
Step 1: 예상 낙찰가 = 감정가(시세) × 해당 지역 평균 낙찰가율
Step 2: 임차인 배분 가능 금액 = 예상 낙찰가 − 선순위 채권 총액
Step 3: 위험 노출 구간 = 보증금 − 임차인 배분 가능 금액
```

- 낙찰가율은 범위로 산출 → 결과도 구간으로 제시
- 등기부 미업로드 시: 선순위 채권 추정 불가 → 전세가율 기반 개략적 위험 수준만 표시
- 면책 문구 상시 표시: "본 추정치는 법적 확정 금액이 아니며, 계약 의사결정 참고용 정보입니다."

#### 분석 단계별 산출물

| 단계 | 필요 입력 | 산출물 |
|------|----------|--------|
| 1단계 (즉시) | 주소 + 보증금 | 전세가율 등급 + 보증 가입 가능성 + 개략적 위험 수준 |
| 2단계 (정밀) | + 등기부등본 PDF | 등기 권리관계 분석 + **위험 노출 구간(금액)** |
| 3단계 (개인화) | + 마이데이터 동의 | 재무 대응 여력 반영 (**Phase 2**) |

### F-06. Lease Passport 리포트

| 항목 | 내용 |
|------|------|
| 기능 ID | F-06 |
| 범위 | **MVP 필수** |
| **입력** | F-05 분석 결과 |
| **처리** | 분석 결과를 3-Layer 리포트로 구성 |
| **출력** | 시각화된 Lease Passport 리포트 화면 |

#### Layer 1 — 종합 판정 + 핵심 수치

| 표시 항목 | 예시 |
|----------|------|
| 종합 판정 | 🟢 계약 가능 / 🟡 보완 후 가능 / 🔴 신중 검토 권고 |
| 위험 노출 구간 | 약 8,500만 원 ~ 1.1억 원 |
| 보증 가입 시 변화 | → 약 ○○만 원 이하로 축소 |
| 면책 문구 | ※ 본 추정치는 법적 확정 금액이 아니며, 계약 의사결정 참고용 정보입니다. |

#### Layer 2 — 항목별 위험 분석

- 분석된 변수별 🟢/🟡/🔴 카드
- 각 카드: 위험 등급 + 근거 데이터 + 한 줄 해석
- 예: "전세가율 82% → 해당 지역 평균(74%) 대비 높음 — 🟡"

#### Layer 3 — 맞춤형 액션 플랜

- 종합 판정에 따라 다른 액션 버튼 세트 (F-07 참조)

#### 부가 기능

| 기능 | MVP | Phase 2 |
|------|-----|---------|
| 앱 내 "내 Passport" 저장 | ✅ | |
| 카카오톡 공유 (요약 카드) | | ✅ |
| PDF 다운로드 | | ✅ |

#### Phase 2 확장: 리포트 해설형 AI Q&A

리포트 결과를 바탕으로 사용자의 후속 질문에 답변하는 설명형 AI 도우미를 제공한다.

| 항목 | 내용 |
|------|------|
| 범위 | **Phase 2 확장** |
| 성격 | 범용 챗봇이 아닌, **리포트 결과 기반 설명형 Q&A** |
| 예시 질문 | "왜 이 매물은 🟡인가요?" / "전세가율이 높다는 게 무슨 뜻인가요?" / "보증 가입을 위해 뭘 보완해야 하나요?" / "지금 가장 먼저 해야 할 행동은?" |
| 응답 범위 | 해당 리포트의 분석 결과·판정 근거·추천 액션에 한정. 법률·세무 상담 등 서비스 범위 외 질문에는 "전문 상담을 예약해보세요" 안내 |
| 설계 원칙 | 서비스 정체성(진단 + 행동 연결)을 유지하면서, 청년층의 정보 비대칭을 추가적으로 해소 |

> 이 기능은 진단 엔진의 보조 확장이지, 서비스의 중심축을 챗봇으로 이동시키는 것이 아니다.

### F-07. 후속 금융 액션 연결

| 항목 | 내용 |
|------|------|
| 기능 ID | F-07 |
| 범위 | **MVP 필수** |
| **입력** | 종합 판정 결과 + 매물 정보 |

#### 판정별 액션

| 판정 | 액션 | 연결 방식 |
|------|------|----------|
| 🟢 | 전세보증금반환보증 가입 안내 | 보증기관 연계 페이지 → 보증료 시뮬레이션 |
| 🟢 | 전세대출 사전 심사 | 하나은행 전세대출 상품 비교 (딥링크) |
| 🟢🟡 | 안전 계약 체크리스트 | 확인서류 목록 화면 제공 |
| 🟡 | 🟡 항목 확인 가이드 | 리포트 내 해당 항목 강조 + 안내 |
| 🟡🔴 | 전문 상담 예약 | 하나은행 주거금융 상담원 예약 (리포트 자동 첨부) |
| 🔴 | 🔴 요인 상세 해설 | 리포트 내 위험 요인 상세 설명 |
| 🔴 | 대안 매물 탐색 안내 | **Phase 2** (부동산 플랫폼 제휴) |

#### 기술적 연결
