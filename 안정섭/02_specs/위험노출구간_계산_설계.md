# 위험 노출 구간 계산 설계

> 실제 법원경매 배당 원리 기반 검증 및 API/업로드 데이터 분류  
> 작성일: 2026-04-13

---

## 1. 기존 공식의 문제점

### 기존 공식
```
예상 낙찰가       = 감정가(시세) × 낙찰가율(65~80%)
임차인 배분 가능  = 예상 낙찰가 - 선순위 채권(근저당)
위험 노출 구간    = 보증금 - 임차인 배분 가능 금액
```

### 문제점 3가지

**① 집행비용 미공제**  
법원경매에서 낙찰가는 배당에 앞서 집행비용(감정평가비, 공고료, 송달료 등)을 먼저 공제한다.  
통상 낙찰가의 **2~3%** 수준이며, 이를 빠뜨리면 배분 가능 금액을 과대 추정한다.

**② 선순위 채권 범위 협소**  
근저당만 선순위 채권으로 보는 것은 부정확하다. 실제 배당 순위는 아래와 같다.

| 배당 순위 | 항목 | 비고 |
|----------|------|------|
| 1순위 | 집행비용 | 낙찰가의 2~3% |
| 2순위 | 필요비·유익비 | 건물 보존을 위한 지출 |
| 3순위 | **소액임차인 최우선변제** + 최종 3개월 임금 | 근저당 설정 시점과 무관하게 우선 |
| 4순위 | **당해세** (재산세, 종부세, 상속·증여세 등) | 등기 순서 무관하게 항상 우선 |
| 4순위 (동) | 법정기일이 빠른 조세채권, **확정일자 임차인 우선변제** | 날짜 기준 경합 |
| 5순위 | 근저당·저당권·전세권 등 담보채권 | 등기 설정일 기준 |

전세사기 피해 사례 대부분에서 **체납 세금(당해세)**이 숨어 있었고, 이것이 임차인 배당을 크게 줄이는 핵심 원인이었다.

**③ 낙찰가율 65~80%는 빌라에 부적합**  
| 주택 유형 | 실제 낙찰가율 (2023~2024) | 비고 |
|----------|--------------------------|------|
| 서울 아파트 | 97~100%+ | 경쟁 심함 |
| 수도권 오피스텔 | 80~90% | 아파트보다 낮음 |
| **서울 빌라(연립다세대)** | **50~70%** | 유찰 반복 후 낮은 가격에 낙찰. 낙찰률 자체가 10~12%로 매우 낮음 |

서비스 주 대상이 빌라·오피스텔이므로 낙찰가율 하한을 **55%**로 조정해야 보수적이고 현실적이다.

## 2. 수정된 계산 공식

### 소액임차인 최우선변제 기준 (2025년)

| 지역 | 소액임차인 보증금 상한 | 최우선변제 한도 |
|------|----------------------|----------------|
| 서울특별시 | 1억 6,500만 원 이하 | 최대 5,500만 원 |
| 수도권 과밀억제권역 | 1억 4,500만 원 이하 | 최대 4,800만 원 |
| 광역시 (과밀 제외) | 8,500만 원 이하 | 최대 2,800만 원 |
| 기타 지역 | 7,500만 원 이하 | 최대 2,500만 원 |

> 단, 전체 최우선변제 합계 ≤ 주택가액의 1/2

---

### 수정 공식

```
[Step 1] 예상 낙찰가 구간 산출
  낙찰가율 범위:
    - 보수적(빌라): 55~65%
    - 기본(오피스텔/아파트): 70~85%
  예상 낙찰가 = 시세 × 낙찰가율

[Step 2] 집행비용 공제
  집행비용 = 예상 낙찰가 × 2.5% (중간값 추정)
  배분 가능 총액 = 예상 낙찰가 - 집행비용

[Step 3] 임차인보다 앞서는 선순위 항목 합산
  선순위 합계 =
      소액임차인 최우선변제금 (건물 내 다른 세대 추정치 포함)
    + 당해세 추정액 (확인 불가 → 보수적 추정값 또는 경고 표시)
    + 근저당 채권최고액 합계        ← 등기부등본에서 추출
    + 선순위 확정일자 임차인 보증금 ← 등기부등본에서 추출 (전세권 설정 시)

[Step 4] 임차인 배분 가능 금액
  배분 가능 금액 = 배분 가능 총액 - 선순위 합계
  (음수인 경우 → 0으로 처리, 전액 손실 가능성 표시)

[Step 5] 위험 노출 구간
  위험 노출 = 보증금 - 배분 가능 금액
  (구간으로 표시: 낙찰가율 min/max 두 시나리오 모두 계산)
```

### 계산 예시 (김지원 케이스, 서울 관악구 빌라)

```
입력값:
  보증금        = 2억 원
  시세          = 2.45억 원
  근저당        = 1.8억 원
  가압류 1건    = 존재 (금액 불명)

[Step 1] 예상 낙찰가
  보수: 2.45억 × 55% = 1.35억
  기본: 2.45억 × 65% = 1.59억

[Step 2] 집행비용 공제
  보수: 1.35억 - 338만 = 1.317억
  기본: 1.59억 - 398만 = 1.550억

[Step 3] 선순위 합계
  근저당:         1.8억
  소액임차인 추정: 5,500만 (서울 최대치 × 세대수 추정)
  당해세:         알 수 없음 → 경고 표시
  선순위 합계:    최소 2.35억

[Step 4] 배분 가능 금액
  보수: 1.317억 - 2.35억 = -1.033억 → 0
  기본: 1.550억 - 2.35억 = -0.800억 → 0

[Step 5] 위험 노출
  보증금 2억 - 0 = 2억 전액 손실 가능
  → 🔴 판정 (전액 손실 시나리오)
```

> **주의**: 당해세(임대인 체납 재산세 등)는 API로 조회 불가.  
> 발표 시 "당해세는 확인 불가한 변수이며, 존재 시 위험도가 더 높아집니다" 명시 필요.

## 3. 데이터 소스 분류

### 원칙
- **API 자동 조회**: 사용자가 아무것도 안 해도 서비스가 백엔드에서 자동으로 가져오는 데이터
- **등기부등본 업로드**: 사용자가 직접 PDF를 업로드하는 단 하나의 입력
- **산출값**: API + 등기부 데이터를 조합해 내부적으로 계산하는 값
- **확인 불가**: 현재 기술적으로 자동화 불가능한 항목

---

### 분류표

| 데이터 항목 | 소스 구분 | API명 / 방법 | 비고 |
|------------|----------|-------------|------|
| **매물 주소 → 표준화 좌표** | API 자동 | 카카오 Local API | 주소 입력 즉시 자동 처리 |
| **아파트 실거래가 (전세/매매)** | API 자동 | 국토부 실거래가 정보 API (data.go.kr) | 법정동 코드 + 계약년월 파라미터 |
| **연립다세대 실거래가** | API 자동 | 국토부 실거래가 정보 API (별도 엔드포인트) | 아파트와 엔드포인트 다름 |
| **오피스텔 실거래가** | API 자동 | 국토부 실거래가 정보 API (별도 엔드포인트) | 위와 동일 |
| **건축물대장** (면적, 용도, 준공일, 세대수) | API 자동 | 국토부 건축HUB 건축물대장정보 서비스 | 세대수로 소액임차인 추정에 활용 |
| **공동주택 공시가격** | API 자동 | 국토부 공동주택가격정보 WFS | 시세 추정 보조값 |
| **개별공시지가** | API 자동 | 국토부 개별공시지가 API | 비아파트 시세 추정 보조 |
| **시세 추정값** | 산출값 | 실거래가 + 공시가격 조합 | 유사 면적·층수 기준 |
| **전세가율** | 산출값 | 보증금 ÷ 시세 | 사용자 입력 + API 조합 |
| **지역별 낙찰가율** | 내부 DB | 한국부동산원 R-ONE 통계 (정기 업데이트) | 공공 API 없음 → 주택 유형별 배치 저장 |
| **HUG 보증 가입 가능 여부** | 내부 룰 | HUG 공개 기준을 룰 엔진으로 구현 | API 없음 → 조건식으로 자체 판별 |
| **소액임차인 최우선변제 기준** | 내부 룰 | 법령 기준 → 지역별 룰 테이블 | 법 개정 시 업데이트 필요 |
| **근저당 건수 / 채권최고액** | **등기부등본 업로드** | OCR + 정규식 파싱 (을구) | registry_ai_logic.py 구현 완료 |
| **가압류 / 압류 / 가처분 유무** | **등기부등본 업로드** | OCR + 정규식 파싱 (갑구) | 동일 |
| **신탁 등기 유무** | **등기부등본 업로드** | OCR + 정규식 파싱 (갑구) | 동일 |
| **소유권 이전 이력 / 날짜** | **등기부등본 업로드** | OCR + 정규식 파싱 (갑구) | 1년 이내 이전 여부 판별 |
| **선순위 전세권 설정 여부** | **등기부등본 업로드** | OCR + 정규식 파싱 (을구) | 현재 코드 추가 필요 |
| **집행비용 추정** | 산출값 | 예상 낙찰가 × 2.5% (고정 추정) | 정밀값 불필요 |
| **당해세 (체납 재산세 등)** | **확인 불가** | 임대인 동의 없이 자동 조회 불가 | → 경고 문구 표시로 대응 |
| **임대인 소득세·부가세 체납** | **확인 불가** | 세무서 직접 신청 필요 | → 상담 연결로 안내 |

## 4. "확인 불가" 항목 대응 방법

당해세와 체납 조세는 현재 기술적으로 자동 조회 불가하다.  
이는 서비스의 한계이지만, **발표에서 약점이 아닌 신뢰 장치**로 프레이밍 가능하다.

```
리포트 내 문구 예시:
"⚠️ 임대인 체납 세금(당해세)은 자동 확인이 불가합니다.
체납 세금이 존재할 경우 위험 노출 구간이 더 커질 수 있습니다.
계약 전 임대인 동의하에 세무서 또는 홈택스에서 확인을 권장합니다.
[전문 상담 예약하기]"
```

## 5. 판정 로직 업데이트 권장사항

현재 등기 권리관계 🔴 판정 기준에 아래 항목 추가 권장:

| 추가 항목 | 현재 코드 반영 여부 | 조치 |
|----------|-------------------|------|
| 선순위 전세권 설정 | ❌ 없음 | registry_ai_logic.py 파싱 로직 추가 필요 |
| 소유권 이전 후 단기 재매도 패턴 | ❌ 없음 | Phase 2 (거래 패턴 이상 변수) |
| 집행비용 공제 | ❌ 없음 | compute_risk_exposure() 함수 수정 필요 |
| 낙찰가율 주택 유형별 분기 | ❌ 없음 | 주택 유형(아파트/빌라/오피스텔) 파라미터 추가 |

## 6. HUG 공식과의 비교

| 항목 | 우리 공식 | HUG 공식 |
|------|----------|----------|
| 기준 가격 | 실거래가 API 기반 시세 추정 | KB시세 / 공시가격×140% / 안심전세 하한가 중 최솟값 |
| 위험 판단 방식 | 낙찰가율 기반 손실 구간 직접 추정 | 보증금+선순위채권 ≤ 주택가격×90% |
| 선순위 채권 | 등기부 파싱 (근저당, 가압류 등) | 동일 (단순 합산) |
| 집행비용 | 별도 공제 (수정 후) | 미반영 |
| 당해세 | 경고 표시 | 미반영 |
| 소액임차인 | 세대수 추정 반영 (수정 후) | 미반영 |
| 결과 형태 | **금액 구간** (보수/기본 시나리오) | 가입 가능/불가 이진 판정 |

**차별점 요약**: HUG는 "가입 가능한가?"를 판단하지만, 우리는 "얼마를 잃을 수 있는가?"를 추정한다.  
HUG 공식보다 더 많은 변수를 반영하므로 발표에서 기술적 우위로 제시 가능.
